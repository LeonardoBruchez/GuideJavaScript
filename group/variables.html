Var,Let e Const

Use const para a maioria das variáveis

  • Declara uma variável cujo valor NUNCA pode ser reatribuído (trocado).
    • const preço = 50;
    • preço = 60;
    • ❌ ERRO!
  • Escopo de Bloco ({}) — A variável só existe no bloco onde foi definida. Isso é seguro e previsível.
  • OBRIGATÓRIA. Precisa ter um valor logo na declaração.

Use let apenas quando você souber que o valor da variável precisará ser alterado no futuro.

  • Declara uma variável cujo valor PODE ser reatribuído (trocado).
  • Escopo de Bloco ({}) — Assim como o const, é moderno e seguro.
  • PROIBIDA redeclaração. Você não pode declarar(criar) uma variável let duas vezes no mesmo escopo.
    • Ex: let x = 1;
    • let x = 2;
    • ❌ ERRO!

O var é a forma antiga de declarar variáveis no JavaScript (antes de 2015). É recomendado EVITAR seu uso em novos projetos.

  • Declara uma variável.
  • Escopo de Função (não respeita blocos {}).
    • Ele "vaza" para fora de blocos if ou for, podendo sobrescrever variáveis que você não pretendia.
  • Redeclaração permitida, você pode declarar a mesma variável várias vezes sem aviso, escondendo bugs.
